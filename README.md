# Multi-Level (Laplacian/Gaussian Pyramid) Full Waveform Inversion

This repository implements **Full Waveform Inversion (FWI)** and provides a comparative study between the conventional **frequency-continuation multi-scale (MS)** strategy and two **image-pyramid-based multi-level FWI** strategies using Laplacian and Gaussian pyramids.
---

## Overview

Full Waveform Inversion is an iterative, gradient-based method for recovering high-resolution
subsurface velocity models from recorded seismic waveforms. Because the FWI objective function is
highly non-convex, strategies that reduce cycle-skipping — such as multi-scale frequency continuation
or multi-resolution image pyramids — are commonly used to guide the optimization toward the global
minimum.

This project explores and compares:

- **Multi-scale FWI (MS)** — the classical frequency-continuation approach, inverting from low to high
  frequencies progressively.
- **Multi-level FWI with Laplacian pyramids** — a coarse-to-fine strategy that decomposes the model/data
  using a Laplacian pyramid.
- **Multi-level FWI with Gaussian pyramids** — a coarse-to-fine strategy using Gaussian pyramid
  smoothing instead.

## Repository Structure

### Notebooks

| Notebook | Description |
|---|---|
| `01_gen_data.ipynb` | Generates the synthetic true/initial velocity models and the corresponding observed seismic data (forward modeling). |
| `02_fwi_ms.ipynb` | Classical multi-scale FWI using frequency continuation (low-to-high frequency inversion). |
| `03_fwi_ml_lap.ipynb` | Multi-level FWI based on a **Laplacian pyramid** decomposition. |
| `04_fwi_ml_gau.ipynb` | Multi-level FWI based on a **Gaussian pyramid** decomposition. |
| `04_fwi_ms_gs.ipynb` | TODO: describe (e.g. multi-scale FWI with grid-search / Gauss–Seidel variant — please confirm). |

### Folders

| Folder | Contents |
|---|---|
| `model_data/` | Model parameters: true velocity model and initial velocity model used for the inversion. |
| `obs_data/` | Observed seismic data generated by forward modeling (used as the inversion target). |
| `source/` | Source time functions / source wavelets. |
| `src_rec_loc/` | Source and receiver geometry (positions). |
| `log_data/` | Inversion logs (e.g. loss curves, iteration records). |
| `rec/` | Reconstructed / inverted velocity models (results). |
| `tools/` | Shared utility functions used across the notebooks. |

## Usage

Run the notebooks in order:

1. **Generate data** — `01_gen_data.ipynb`
   Builds the true/initial velocity models and simulates observed data.

2. **Run inversion** — choose one of:
   - `02_fwi_ms.ipynb` (classical multi-scale)
   - `03_fwi_ml_lap.ipynb` (Laplacian-pyramid multi-level)
   - `04_fwi_ml_gau.ipynb` (Gaussian-pyramid multi-level)
   - `04_fwi_ms_gs.ipynb` (TODO)

3. **Inspect results** — inverted models are saved to `rec/`, and inversion logs (loss/error curves) are
   saved to `log_data/`.
